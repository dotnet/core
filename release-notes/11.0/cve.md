# .NET 11 CVEs

The .NET Team releases [monthly updates for .NET 11](https://github.com/dotnet/announcements/labels/.NET%2011.0) on [Patch Tuesday](https://en.wikipedia.org/wiki/Patch_Tuesday). These updates often include security fixes. If you are on an older version, your app may be vulnerable.

Your app needs to be on the latest .NET 11 patch version to be secure. The longer you wait to upgrade, the greater the exposure to CVEs.

## Which CVEs apply to my app?

Your app may be vulnerable to the following published security [CVEs](https://www.cve.org/) if you are using an older version.

No CVEs have been published for .NET 11 yet. This page will be updated when .NET 11 reaches GA.

The CVE exposure is cumulative. For example, `11.0.0` users may be vulnerable to the CVEs present in `11.0.0` and newer releases. Similarly, `11.0.1` users may be vulnerable to the CVEs present in `11.0.2` and newer releases. The latest release is not vulnerable to any published CVEs.
